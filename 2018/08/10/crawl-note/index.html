获取动态页面

  • 网页通过动态加载信息
  • 无法直接通过html页面直接获取信息时
    • 通过向服务器发送请求获取信息
    • 通过浏览器帮助获取信息
  • 服务器与爬虫交互时，使用都是字节串的形式
  • 需要通过urlencode() 对url后再传递给服务器

  • 接收到服务器的信息后，需要对信息进行解码

    该网站可进行 解码 和 编码 的操作
http://tool.chinaz.com/tools/urlencode.aspx

get方法

将参数拼接到目标url的后面
from urllib import parse
\# 以字典的形式保存参数
a = {'a':12, 'b':'编码'}
ss = parse.urlencode(a)
\# ss = 'a=12&b=%E7%BC%96%E7%A0%81'
print(ss)

selenium 库爬取动态页面

  • selenium 常用于自动化测试
  • PhantomJS 无界面的Firefox浏览器，节省资源

模拟用户操作浏览器获取信息

  1. 下载chromedriver ，在Python与浏览器之间搭建一个桥梁
    • geckodriver—-firefox浏览器
  2. 将chromedriver放入python解释器统一路径
  3. 进行操作

  4. 关闭浏览器

    • 浏览器使用完毕后，需要关闭，此时python给浏览器发送一个信号

    • 但浏览器在运行其他任务时，不会关闭，在必要的时候kill 此进程

      from selenium import webdriver

\# 创建chrome对象
driver = webdriver.Chrome()
driver.get('http://cn.bing.com')
a = driver.find_element_by_id('sb_form_q').send_keys('python爬虫')
b = driver.find_element_by_id('sb_form_go').click()
\# a = driver.find_element_by_xpath('//[@id="sb_form_q"]')
print(driver.page_source)

driver.quit()
driver.close()
  • 在实际应用场景中，使用selenium 方法消耗时间太多
    • 浏览器需要加载太多资源包，而我们并不需要

  • 浏览器启动需要大量资源，且需要另外的管理方式

    • 不到万不得已，尽量不使用此方法

  • 对浏览器进行基础设置

    \# 加启动配置
\# 忽略‘Chrome正在受到自动软件的控制’
option = webdriver.ChromeOptions()
option.add_argument('disable-infobars')
return webdriver.Chrome(chrome_options = option,desired_capabilities = None)
\# 打开chrome浏览器
driver = webdriver.Chrome(chrome_options=option)
driver.get("https://www.baidu.com") 

二、启动浏览器并加载浏览器的静默模式，让它在后台运行。用headless

from selenium import webdriver
\# 加启动配置
option = webdriver.ChromeOptions()
option.add_argument('headless')
