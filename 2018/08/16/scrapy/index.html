scrapy基础使用

  1. 安装

    使用anaconda 命令 conda install scrapy
安装完毕后，在命令行中，测试 scrapy 是否能够正续航运行

  1. 创建项目

    1. 进入文件夹，使用cmd 命令行执行

      scarpy startproject <projectname>

    2. 进入创建好的文件夹 

      cd projectname

    3. 创建 spider

      scrapy genspider <spidername> <domain>

    4. 运行spider

      scrapy crawl <spidername>
# 或者  scrapy runspider + spidername.py的路径(运行自包含在一个py文件中的爬虫，不需要创建项目)
scrapy runspider spiders/spidername.py

    5. spider 默认使用get方法请求

      1. 使用 表单post请求 

2. 发送 json post请求




3. 发送更多请求，使用 `scrapy.Request` 方法，查看参数

scrapy 运行逻辑

1. 运行主要命令通过 终端命令行来控制，终端会显示 scrapy爬取时的一些信息，和日志文件
2. 每个模块的功能划分
    1. spider 负责访问网页得到response，并解析数据
    2. item 负责数据 的结构化保存， 文件，数据库的连接
    3. pipeline 负责数据的过滤，去重
    4. setting 负责整个scrapy的参数设置
3. spider 将数据爬取下来，传递给pipeline过滤，过滤后传递给 item 进行存储

scrapy 命令行大全

命令 用法
scrapy -h 查看所有命令
scrapy version 查看版本号
scrapy version -v 查看详细信息
scrapy startproject 创建新项目
scrapy genspider url 创建新爬虫，一个项目中可以有多个spider
scrapy list 查看项目中的所有爬虫
scrapy view http://www.baidu.com 使用浏览器打开网页，网页是请求发送后，保存到本地
再通过浏览器打开，可以分析网页是否为ajax加载
scrapy check 检查代码是否有误
scrapy shell 进入scrapy shell 命令，主要用来测试
shell –> response.xpath(‘//*[@id=xx]’) 使用shell 中的response 进行操作
shell runspider spiders/xxspider.py -o file.json(file.csv) 运行并保存数据
shell crawl -o file.json(file.csv) 运行并保存数据

The different between crawl and runspider are crawl use ,spider use file path of the spider!

在 crawl 和 runspider 之间的区别是， runspider 使用的 spider 的文件路径 命令用于直接运行创建的爬虫, 并不会运行整个项目
而 crawl 使用的是 spider的名称

crawl 命令

# 在命令行中传入参数
scrapy crawl spider -a 变量名=变量

# 将爬取信息输入文件
scrapy crawl spider -o filename.format

# 限制爬虫爬取item数量，爬取到10 个 item 后停止爬虫 
scrapy crawl spider -s CLOSESPIDER_ITEMCOUNT=10

# 对文件的格式进行设置
scrapy crawl spider -o a.json -t json

genspider 命令

# 使用模板生成模板，在templates中的spiders中
scrapy genspider [-t template] <name> <domain>

# 基础使用, 生成spider

scrapy genspider <name> <domain>

# 查看所有模板

scrapy genspider -l
