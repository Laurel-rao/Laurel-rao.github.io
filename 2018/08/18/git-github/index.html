git 是一个分布式版本控制系统

特点 集中式 分布式
系统特点 1.集中式系统，版本库集中存放于中央服务器
2.企业内并行集中开发
3.windows推荐使用
4.克隆大量文件时，速度慢		 1.分布式系统，版本库保存在所有人的电脑里
2.开源项目开发
3.mac，linux上推荐使用
4.克隆大量文件时，速度快
灵活性 中央服务器出现故障，所有操作无法执行 1.git服务器出现故障，不影响本机操作
2.可查看本地日志，效率较高
备份 体验较差，定期备份，整个svn实行完全备份 每台电脑都拥有版本库，gitlab集成了备份功能
版本控制 保存文件的差异数据，并生成一个全局版本号 1.git将40位的hash值作为版本号
2.git rebase操作可以更好的保持提交记录的整洁
3.保存文件的快照
分支方面 1.拉分支更像是copy一个路径
2.可针对任何子目录进行branch
3.拉分支的时间较慢，因为拉分支相当于copy
4.创建完分支后，影响全部成员，每个人都会拥有这个分支
5.多分支并行开发较重（工作较多而且繁琐）
 1.我可以在Git的任意一个提交点（commit point）开启分支！（git checkout -b newbranch HashId）
2.拉分支时间较快，因为拉分支只是创建文件的指针和HEAD
3.自己本地创的分支不会影响其他人
4.比较适合多分支并行开发
5.git checkout hash值(切回之前的版本，无需版本回退)
6.强大的cherry-pick

代码版本控制

  • 记录文件

    • 修改者，被修改的内容，时间，修改备注

  • 常见问题
    -fatal: refusing to merge unrelated histories

    - 无法正常 pull ，使用下面这条语句
        git pull origin master --allow-unrelated-histories
    • ! [rejected] master -> master (non-fast-forward)
      • 使用 pull拉取后重试
    • warning: LF will be replaced by CRLF i
      • 文件中因为 LF 与 CSLF 的不同，无法 add
        git config core.autocrlf false
  • 分支系统

git 命令

命令 说明
init 初始化仓库
add file
add file -A 提交所有文件及文件夹
commit -m ‘备注’ 提交并对修改内容作备注
log 查看文件版本
status 查看缓存据状态
diff 查看缓存区与工作区的差异
diff – cached 查看暂存区和历史区的差异
diff master 查看历史区与工作区的差异
git checkout index.html 撤回内容
git reset HEAD index.html 取消add到缓存区的内容
rm filename 删除文件
rm -rf 强制删除
reset –hard HEAD/ commit-id 回滚到 某一版本

分支管理

命令 说明
git branch dev 创建分支
git checkout dev 切换到dev分支
git checkout -b dev 创建分支并切换到该分支
git branch -d dev 删除分支
git branch -d -f dev 删除分支
git branch -D dev 强制删除分支
git push origin –delete resume 删除远程分支
git commit -am ‘备注’ 在分支上提交
git merge 本分支名 次分支名 合并分支

远程仓库

命令 说明
git remote add origin git@github.com:michaelliao/learngit.git 关联远程与本地仓库
git pull origin master 拉取远程仓库文件
git push -u origin master 第一次上传文件
git push origin master 普通上传文件
git clone xxx.git 将远程仓库上的文件下载下来
git branch -m 分支名 创建分支并切换到该分支
