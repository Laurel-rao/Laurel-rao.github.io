MongoDB

  1. MongoDB数据库（非关系型数据库中的文档型数据库）

    1. 由C++编写的数据库管理系统，因为\0在C++中有特殊的意义，所有不能使用该字符
    2. 支持丰富的增删改查操作
    3. 支持丰富的数据类型
    4. 支持众多的编程语言接口
    5. 使用方便，便于部署，在非关系数据库中属于比较成熟的数据库

  2. 安装mongoDB(默认端口号 27017)

    • 官网自行百度

MongoDB使用

  1. 创建(切换)数据库 use databasename

    • use实际功能是表示选择使用哪个数据库，当这个数据库不存在时即表示创建该数据库
    • 使用use后数据库并不会马上被创建，而是需要插入数据后数据库才会创建

  2. 查看数据库 show dbs , 查看当前数据库 db

  3. 数据库名称规则

    1. 原则上是任意满足以下几条的utf-8字符
    2. 不能是空字符，不能含有空格’ ‘ 点’.’ ‘/‘ ‘\’ ‘\0’
    3. 习惯上使用英文小写
    4. 长度不超过64字节
    5. 不能使用 admin local config 这样的名字

  4. 本地原始数据库

    1. admin ： 存储用户
    2. local ： 存储本地数据
    3. config ： 存储分片配置信息

  5. mongo 数据库db变量说明

    1. db ： mongo系统全局变量 代表你当前正在使用的数据库
    2. db 默认为test 如果插入数据即创建test数据库

  7. 数据库的监测命令(命令行下)

    • mongostat

    • db.createCollection(collection_name)

  10. 查看数据库中集合

    • show tables

  11. 集合的命名规则：

    1. 不能为空字符串，不能有’\0’
    2. 不能以 system.开头 这是系统集合的保留前缀
    3. 不能和保留字重复

数据操作说明

  • 特别说明

  • mongo 名词

    • 文档中键 —–》 域/字段
    • 文档 ———》 记录

  • 集合中文档特点：

    1. 集合中的文档域不一定相同 —》不保证数据一致性
    2. 集合中的文档结构不一定相同

  • 集合设计原则：
    1.集合中的文档尽可能描述的数据类似
    2.同一类文档放在相同的集合，不同的文档分集合存放
    3.层次的包裹不宜太多

  • 插入文档

    • 当作为文档插入时键可以不加引号
    • 查看文档时，需要加引号

  • 查看插入结果 db.class0.find()

  • 插入多条文档 db.collectionName.insert([{},{},{}])

  • save 插入数据 db.collectionName.save()

  • 非关系型数据库和关系型数据库的比较

    1. 不是以关系模型构建数据结构，结构比较自由
      不保证数据的一致性
    2. 非关系型数据库弥补了关系型数据库的一些不足，能 够在处理高并发，海量数据上体现优势
    3. 非关系型数据库的个性化使其可以在节省空间，提高 效率方面发挥作用

  1. 向数组中添加一项，不能和其他的项重复 $addToSet

  2. 时间函数(时间需要加上引号)

    2. new Date() ,自动生成当前时间
      • db.class0.insert({title:'Python入门',date:new Date()})
    3. Date() ,将生成的当前时间变为字符串
      • db.class0.insert({title:'Python疯狂',date:Date()})

    4. ISODate(str)

      • 功能：生成mongo时间存储类型

      • 参数：参数为指定时间

      1. valueof() ,将当前时间生成为时间戳

        • db.class0.insert({title:'Python放生',date:ISODate().valueOf()})

null

  1. 如果某个域存在却没有值可以设置为null

  2. 某个域不存在，可以使用null来匹配

Object 内部文档

  • 文档内某个域的值还是一个文档则这个文档为内部文档

  • 当需要使用内部文档某个域时，可以使用外部文档 . 引用内部文档的方式。但是注意在使用时需要加引号

数组的下标引用

  • 使用数组时，可以使用数组域 . 数组序列下标的方式使用数组中的具体某一个元素
  • 查找结果的有序性
    • 即可以对find查找结果使用[]引用序列下标的方式获取查找结果的第几项
    • db.class2.find({},{_id:0})[3]

索引

  1. 指建立指定键值及所在文档中存储位置的对照清单。使用索引可以方便我们进行快速查找，减少遍历次数，提高查找效率

  2. ensureIndex()

    • 功能： 创建索引

    • 参数： 索引域，索引类别，索引选项

  3. 查看集合索引 db.class1.getIndexes()

    • _id域是由系统自动为集合创建的索引

  4. 自定义索引名称

    • 通过第二个参数中name选项
    • db.class1.ensureIndex({'age':1},{name:'myIndex'})

删除索引

  1. dropIndex（）

    • 功能：删除索引

    • 参数：要删除的索引，可以是索引名称或者索引键值对

  2. dropIndexes()

    • 功能：删除所有索引

索引类型

  1. 复合索引, 根据多个域创建一个索引

    • db.class1.ensureIndex({name:1,age:-1})

  2. 数组和子文档索引

    • 如果对某个域的值为数组或者子文档的域创建索引，则通过数组或者子文档进行查找时也是索引查找

  3. 唯一索引

    • 当将要创建的索引有重复值时，无法创建，并报错
    • 创建索引时希望集合中创建索引的域的值均不重复

    • 当对某个域创建了唯一索引则无法插入重复值

  4. 稀疏索引

    • 只针对有指定域的文档创建索引表，如果某个文档没有该域则不会在索引表中插入内容

  5. 索引约束

    1. 当数据发生更新索引表也要随之更新，影响插入，删除,修改操作效率
    2. 索引表也需要占有一定的磁盘空间

  6. 综上：
    1.当对数据库进行大量的查找操作, 而不是修改操作
    2.集合中文档的数量较大的时候, 比较适合创建索引
