django 中的 FBV 和 CBV

  • FBV function basic views

    • 使用函数进行处理相应的url

      def student(request):
    return HttpResponse('student')

  • CBV class basic views

    • 使用类进行处理响应的url

      # urls.py
url(r'^students/', views.StudentsView.as_view())

# views.py
from django.views import View
class StudentView(View):
    def get(self, request, *args, **kwargs):
        return HttpResponse('GET')

    def post(self, request, *args, **kwargs):
        return HttpResponse('Post')

  • 在生产环境下，静态文件一般直接由 Nginx 等反向代理服务器处理，而应用服务器专心负责处理它擅长的业务逻辑

  1. django 中间件
    • process_request
    • process_view
    • process_response
    • process_exception
    • process_template_response

  2. 中间件的功能

    • 权限
    • 用户登录验证
    • csrftoken 是如何实现的？
      • 在process_view方法
        • 检查视图是否有 @ csrf_exempt 或 @ csrf_protect
        • 去请求体或 cookie1中获取 token 加密字符串

  3. 给 CBV 通过 csrf认证

    1. 重写 dispatch 加csrf 装饰器

      @method_decorator(csrf_exempt)
def dispatch(self, request, *args, **kwargs):
    return super(IndexView, self).dispatch(request, *args, **kwargs)

    2. 在类上指明 csrf装饰器

      @method_decorator(csrf_exempt, name='dispatch') # name 表示为dispatch函数装饰
class IndexView(View):

    3. 在类方法上指名装饰器(此方法不可行，依然 403)

      @method_decorator(csrd_exempt)
def post(request, *args, **kwargs):

  4. 总结：

    1. 本质：基于反射实现
    2. 流程：路由，view， dispatch （反射）
    3. 取消 csrf 认证（装饰器 要加到 dispatch 方法上加 method_decorator 装饰）
    • 扩展：
      • csrf
        • 基于中间件的process_view方法
        • 基于装饰器对单独函数进行设置

restful 规范(建议)

1. API 与 用户的通信协议，总是使用 HTTPs协议
2. method的不同
    1. 对于一个订单的增删改查，只使用一个views 函数
    2. 根据method 的不同，进行不同的操作
3. 域名
    1. https://api.example.com    存在 跨域问题
    2. https://example.com/api    简单
4. 版本
    1. URL ， 如https://api.example.com/v1   （推荐）
    2. 请求头， 在请求头中加入版本号
5. 面向资源编程
    1. 使用名词，对资源进行访问
    2. https://www.example.com/api/v1/名词/
6. 使用状态码与 code 共同使用，描述本次响应的具体信息,
    {
        ‘code’：1000
    }
    - 发生错误时，响应 error 字段
    {
        'error': ‘请求错误’
    }
7. 通过page 和 limit 对 数据进行请求
    http://api.example.com/users?page=1&limit=10
8. 使用过滤来访问
    http://api.example.com/users?id=1

restful 应用

基础配置 restframe官网 http://www.django-rest-framework.org/tutorial/quickstart/

  1. 安装restfulframework 和相应配置

    • pip安装

      pip install djangorestfulframework
pip install markdown
pip install django-filter

    • 或者 git安装

      git clone git@github.com:encode/django-rest-framework.git

  2. 在 install_app 安装 restful

    INSTALLED_APPS = (
    ...
    'rest_framework',
)

  3. 这个步骤不是必须的，需要以网页的形式查看api时, 需要添加 rest 框架的 登录和注销视图

    • 在 根下的 urls 文件添加以下内容

      urlpatterns = [
    ...
    path('api-auth/', include('rest_framework.urls'))
]

  4. 将rest框架的配置，添加到settings.py 中

    REST_FRAMEWORK = {
    # Use Django's standard `django.contrib.auth` permissions,
    # or allow read-only access for unauthenticated users.
    'DEFAULT_PERMISSION_CLASSES': [
        'rest_framework.permissions.DjangoModelPermissionsOrAnonReadOnly'
    ]
}

  5. 创建一个urls文件， 快速开始项目

    from django.conf.urls import url, include
from django.contrib.auth.models import User
from rest_framework import routers, serializers, viewsets

# Serializers define the API representation.
class UserSerializer(serializers.HyperlinkedModelSerializer):
    class Meta:
        model = User
        fields = ('url', 'username', 'email', 'is_staff')

# ViewSets define the view behavior.
class UserViewSet(viewsets.ModelViewSet):
    queryset = User.objects.all()
    serializer_class = UserSerializer

# Routers provide an easy way of automatically determining the URL conf.
router = routers.DefaultRouter()
router.register(r'users', UserViewSet)

# Wire up our API using automatic URL routing.
# Additionally, we include login URLs for the browsable API.
urlpatterns = [
    url(r'^', include(router.urls)),
     url(r'^api-auth/', 
     include('rest_framework.urls', namespace='rest_framework'))
]
