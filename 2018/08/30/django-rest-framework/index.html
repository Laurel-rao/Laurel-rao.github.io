  3. 给 CBV 通过 csrf认证

    1. 重写 dispatch 加csrf 装饰器

      @method_decorator(csrf_exempt)
def dispatch(self, request, *args, **kwargs):
    return super(IndexView, self).dispatch(request, *args, **kwargs)

    2. 在类上指明 csrf装饰器

      @method_decorator(csrf_exempt, name='dispatch') # name 表示为dispatch函数装饰
class IndexView(View):

    3. 在类方法上指名装饰器(此方法不可行，依然 403)

      @method_decorator(csrd_exempt)
def post(request, *args, **kwargs):

    4. 多个装饰器时

      - 错误写法
@method_decorator(login_required, csrf_exempt)
def post(request, *args, **kwargs):

- 正确写法
@method_decorator(login_required)
@method_decorator(csrf_exempt)
def post(request, *args, **kwargs):

  1. 总结：

    1. 本质：基于反射实现
    2. 流程：路由，view， dispatch （反射）
    3. 取消 csrf 认证（装饰器 要加到 dispatch 方法上加 method_decorator 装饰）
    • 扩展：
      • csrf
        • 基于中间件的process_view方法
          • 检查视图是否被 csrf_exempt 装饰 （免除认证）
          • ？？？？ 那 @csrf_protect呢, 如何实现的
            • 源码剖析，csrf_protect 即时 csrf中间件被注释了，依然能够起到保护作用
            • csrf 对该函数进行中间件引入，使得中间件的process_view能够作用在该函数上
        • 基于装饰器对单独函数进行设置

restful 规范(建议)

1. API 与 用户的通信协议，总是使用 HTTPs协议
2. method的不同
    1. 对于一个订单的增删改查，只使用一个views 函数
    2. 根据method 的不同，进行不同的操作
3. 域名
    1. https://api.example.com    存在 跨域问题
    2. https://example.com/api    简单
4. 版本
    1. URL ， 如https://api.example.com/v1   （推荐）
    2. 请求头， 在请求头中加入版本号
5. 面向资源编程
    1. 使用名词，对资源进行访问
    2. https://www.example.com/api/v1/名词/
6. 使用状态码与 code 共同使用，描述本次响应的具体信息,
    {
        ‘code’：1000
    }
    - 发生错误时，响应 error 字段
    {
        'error': ‘请求错误’
    }
7. 通过page 和 limit 对 数据进行请求
    http://api.example.com/users?page=1&limit=10
8. 使用过滤来访问
    http://api.example.com/users?id=1

6. api的版本的使用

之前公司没有交保护费，用的是http，本来是要用https的，还有用专用域名的时候，前端发送请求出现了问题

状态码跟code一起使用，请求失败发送错误信息等等

restful 应用

基础配置 restframe官网 http://www.django-rest-framework.org/tutorial/quickstart/

  1. 安装restfulframework 和相应配置

    • pip安装

      pip install djangorestfulframework
pip install markdown
pip install django-filter

    • 或者 git安装

      git clone git@github.com:encode/django-rest-framework.git

  2. 在 install_app 安装 restful

    INSTALLED_APPS = (
    ...
    'rest_framework',
)

  3. 这个步骤不是必须的，需要以网页的形式查看api时, 需要添加 rest 框架的 登录和注销视图

    • 在 根下的 urls 文件添加以下内容

      urlpatterns = [
    ...
    path('api-auth/', include('rest_framework.urls'))
]

  4. 将rest框架的配置，添加到settings.py 中

  5. 创建一个urls文件， 快速开始项目

restful 嵌套在已有项目中

  1. 用户认证(Authentication)

  1. 配置认证类

    1. 在 views 中创建属性

      • authentication_classes = [MyAuthentication, ]

    2. 在 settings文件中配置

    - 默认全局使用该认证
    - 当一些类不需要认证时(免认证)，在类中设置authentication_classes 为空
        

    - 这样该类就不需要认证了

3. 当有多个认证类时， 最后一个类返回 (request.user, request.auth),其他类都返回 none，否则后面的类都无法执行
  1. 参数说明表

views > settings > rest_framework 默认的

一般都需要两个方法，一个为判断方法，一个为失败时的处理方法

  • 认证源码流程

    1. dispatch
      • 封装 request
        • 获取定义的认证类(全局、局部)， 通过列表生成式创建对象
      • initial
        • perform_authentication
          • 循环执行 authenticate 方法
            • 返回None，不处理，交给下一认证类处理
            • raise AUthenticationFailed(“认证失败”)，抛出异常表示认证失败

  • 权限源码流程

    1. permission_classes = [Mypremission,]
      • 在Mypermission(继承自 BasePermission)中实现 has_permission方法
      • 返回 True则通过，返回 False 则 禁止访问

  • 访问节流

    1. throttle_classes = [Mythrottle,]

      • 在Mythrottle（继承自BaseThrottle）中实现（allow_request）
      • 返回 True 则继续访问， 返回 False 则 禁止访问
      • wait方法，返回还剩多少秒才能访问，秒为单位
