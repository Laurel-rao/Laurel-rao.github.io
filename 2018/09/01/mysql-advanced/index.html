1. 视图

  • 视图介绍

    • 视图是一种抽象行为，将查询的结果作为一张表进行存储，叫做视图
    • 视图是一种面向函数的行为，将经常查询的表格创建视图存储下来
      • 可以提升查询的效率，因为视图已经在缓存层面进行存储，所以不需要解析树解析
      • 减少一大段的sql 语句的编写，减少出错的概率

  • 查看视图

    show table status where comment = 'view'；

  • 创建视图

    create view xxx as select * from table;

  • 查看视图内容

    select * from xxx;

  • 作为条件

    select * from table where id = (select * from xxx);

选项 说明
fields terminated by ‘字符’ 字段分隔符，默认字符为制表符’\t’

2. bug解决    
    1. mysql 有一个关于文件备份的设置`secure_file_priv`
    2. 使用 `show variables like "%secure%"; `查看
        1. 当 值为 null 时，表示文件导入导出都被禁止
        2. 值为 一个路径是，表示该路径下为安全路径，可以导入导出
    3. 使用 set secure_file_priv = ‘/xxx’， 你会发现它报错了，原因是因为他是只读变量
        

    4. 所以，正确的方式是到配置文件中修改

        > 比较坑爹的是，Mac os 居然没有配置文件，然后百度了一圈，从网上复制了文件，保存下来

        1. 修改之前你需要了解下面几个问题
            1. 配置文件叫啥，他又放在哪里
                1. win ==>  my.ini 被放在安装目录的根目录中
                2. linux/ mac  ==> my.cnf 被放在/etc/my.cnf 或者 /etc/mysql/my.cnf

            2. 配置文件格式是啥，大致有用的配置又该如何修改呢

                1. 配置文件一般有三部分组成

                    

                2. 配置文件的几个部分介绍

                    - [mysqldump] # 关于 mysqldump的设置
                    - [mysql]  #
