1. 视图

  • 视图介绍

    • 视图是一种抽象行为，将查询的结果作为一张表进行存储，叫做视图
    • 视图是一种面向函数的行为，将经常查询的表格创建视图存储下来
      • 可以提升查询的效率，因为视图已经在缓存层面进行存储，所以不需要解析树解析
      • 减少一大段的sql 语句的编写，减少出错的概率

  • 查看视图

    show table status where comment = 'view'；

  • 创建视图

    create view xxx as select * from table;

  • 查看视图内容

    select * from xxx;

  • 作为条件

    select * from table where id = (select * from xxx);

子查询

  1. 定义

    1. 嵌套在其他查询中的查询， 通常使用 where 语句 和 in 搭配()使用
    2. 当嵌套层数过深时，查询性能会低

  2. 使用举例

    select * from a where age in (select age from b);

5. 常用函数

  1. 按照拼音排序 （convert）

    order by convert(name using gbk)；

  2. 查询时拼接字段(concat)，英文(concatenate)

    select concat(name, "--", age) from ...

  3. 使用正则表达式进行匹配(regexp)

    1. 使用说明
      1. 正则表达式，相较于模糊查询 like，功能更强大，但性能较低
2. 使用举例

```
where name regexp "bob|maria";
```    
3. 正则语法
    1. “a|b” 表示  or ,a or b
    2. [0,1,2] == [0-2], 表示其中任意一个
    3. "a{1,2}", 表示 前面字符的数量， 一个或两个a
    4. “a{1, }”, 表示 前面字符至少有一个
    5. . 表示除空格字符外的所有字符  
    6. * 表示前面字符数量 为 0 或 n个
    7. ? 表示前面字符数量 为 0 或 1个 等同于{ ,1}
    8. + 表示前面字符数量 为 1 或 n个 等同于{1, }
    9. \\ 表示 转义字符 \\, 查找"\" 应表示为 “\\\”

  1. 将时间戳转换为 字符串形式（ from_unixtime）

    update a set a_time = from_unixtime(b_time);
