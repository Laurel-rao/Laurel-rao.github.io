docker

Posted on 2018-09-03 | In docker

1. 安装 docker

  1. 版本要求，查看各自系统最低支持版本 uname -r
    1. ubuntu 3.10
    2. centos 6.5以上

  2. 安装

    wget -qO- https://get.docker.com/ | sh

  3. 开启docker服务

    sudo service docker start

  4. 修改docker权限，不需要加 sudo 也可执行(非必需)

    cd /var/run/docker
sudo groupadd docker
sudo gpasswd -a ${USER} docker
newgrp - docker

  5. 测试是否安装成功

    docker run hello-world
  6. 修改dockerhub 源（国外网比较慢，使用国内镜像速度快）
    • docker pull 国内网络链接失败或很卡慢，一般都需要更换至国内 
创建或修改 /etc/docker/daemon.json 文件，修改为如下形式

{
    "registry-mirrors": [
        "http://hub-mirror.c.163.com"
    ],
    "insecure-registries": ['']
}

  1. 以下是国内加速地址

    • Docker中国区官方镜像(http://www.docker-cn.com/registry-mirror) 

      https://registry.docker-cn.com

    • 网易 

      http://hub-mirror.c.163.com

    • ustc 

      https://docker.mirrors.ustc.edu.cn

    • 阿里(这个地址没验证过) 

      https://kfwkfulq.mirror.aliyuncs.com

  2. 重启docker 服务，让修改生效

    • sudo service docker restart

2. docker三要素

  1. 镜像
    1. 镜像是一个能够完整运行的小系统
    2. 镜像包含了操作系统，配置环境，应用程序
    3. 镜像就像一个模板，能够造出很多容器

  2. 容器

    1. 容器是镜像的实体，作为一个正在运行的一个程序
    2. 容器之间相互隔离，是相互独立的进程

  3. 仓库

    1. 仓库是存放大量镜像的地方
    2. 使用 pull 和 push 进行提取
    3. 通过仓库进行镜像的备份和镜像的部署

3. docker 原理

  1. 精简版的虚拟技术
  2. 使用多层镜像的技术，包含应用必须的一些运行环境
  3. 在其他软件中，可以共享这些环境
  4. docker 使用go 语言开发
  5. 优势
    1. 启动速度快，与linux 共用内核，内核级的虚拟化， 实现更少的资源更高的性能
    2. 容器与镜像的灵活程度高，可以随意搭配
    3. 将docker 镜像搭配好，整个运行环境就固定了，不需要额外配置，方便简单，避免运维人员和开发人员之间的部署问题
    4. 使用docker 镜像配置集群的速度更快，迁移和扩展更方便

4. docker 常用命令

  1. 帮助命令

    1. docker info
    2. docker version

    3. docker –help

      1. 查看具体命令

        docker –help

  2. 镜像命令

    1. docker images
      1. 查看镜像
      2. 参数
        1. -a 列出所有镜像(包含中间层)
        2. -q 列出所有镜像的 images ID
        3. –digests 查看镜像的备注信息（摘要）
        4. –no-trunc 查看完整的镜像信息

    2. docker search

      1. 在 docker hub 上搜索 镜像

      2. options

        1. –no-trunc 显示完整镜像信息

        2. -s 列出收藏数不小于 指定值的镜像

          docker search -s 30 tomcat
        3. –automated 只列出autimated bulid 类型的镜像

    3. docker pull

      1. 拉取镜像

    4. docker rmi 

      1. 删除镜像（删除前，需要停止镜像服务）
2. options
    1. -f 强制删除
    2. 删除多个 -f 镜像1:TAG 镜像2:TAG
    3. 删除全部 -f $(socker images -qa)

  3. 容器命令

    1. 运行命令 run container: ‘‘
      1. –name=‘containers’ 为容器制定一个名称
        1. -d 后台运行容器，并返回容器 ID， 即 启动守护式容器
        2. -i 以交互模式运行容器，通常 与 -t 同时使用
        3. -t 为容器重新分配一个伪输入代码， 通常与 -i 同时使用
        4. -P 随机端口映射
        5. -p 指定端口映射，有以下四种格式
        6. ip:hostPort:containerPort
        7. ip::containerPort
        8. hostPort:containerPort
        9. containerPort
    2. docker ps 查看当前运行的容器
      1. -a 历史所有运行的容器
      2. -l 查看上一个运行的容器
      3. -n 查看前n 个 容器信息
      4. -q 只显示容器编号
    3. 退出容器
      1. exit 退出并关闭容器
      2. ctrl + P + Q 退出但并不关闭容器

    4. 启动容器

      1. start container ID

    5. 关闭容器

      1. stop 停止
      2. kill 强制停止

    6. 删除已停止的容器

      1. rm
        1. -f 强制删除，包括正在运行的容器
        2. 删除所有容器
          1. docker rm -f $(docker ps -qa)
          2. docker ps -a -q | xargs docker rm

    7. 启动守护式容器 run -d 容器名

      1. 该容器没有前台进程时，启动后会立即退出
      2. 需要为该容器建立一个死循环，防止进程退出

    8. 查看容器日志 logs -f -t –tail 容器ID

      1. options
        1. -t 加入时间戳
        2. -f 跟随最新的日志打印
        3. –tail 数字显示最后多少条

    9. 查看容器内运行的进程

      1. docker top

    10. 查看容器内部细节

      1. docker inspect
      2. 以 json 格式描述 docker 容器的细节

    11. 进入正在运行的容器并以命令行交互

      1. docker exec -it bashShell
      2. 重新进入docker attach

      3. 两者区别

        1. attach 进入命令行执行
        2. exec 可以直接以 命令行的形式 执行并返回结果，并不进入命令行

        3. exec 也可以 进入命令行执行

          exec -it xxx /bin/bash

    12. 将数据从容器内转换到 宿主机器上

      1. docker cp :/xxx 《目标路径》

    13. 将容器提交为镜像

      1. commit -a ‘作者名’ -m ‘备注’ 容器名：

5. 容器 卷

  1. 容器卷是在容器与主机，容器与容器之间进行交换的共享空间
  2. 使用run -it -v /主机目录:/容器目录 容器
  3. 容器之间进行共享 –volumes-from <容器名>
    1. run -it –name v1 –volumes-from <容器名> 容器
    2. 既可以对容器卷进行共享，容器卷的生命周期会一直存在，直到没有容器使用为止
  4. 容器到主机
    1. docker cp /xx /xx 容器
  5. docker inspect 容器 查看容器具体信息

6. dockerfile

  1. 一种脚本， 用来构建docker 镜像的构建文件
  2. 构建步骤
    1. 编写 dockerfile文件，
    2. docker build 构建文件，生成一个镜像

docker build -f file -t name:tag .
注意 后面有个点

    3. docker run 创建一个容器并运行
3. dockerfile 语法
    1. 每条指令必须为大写字母，指令后面至少跟随一个参数
    2. 指令按照从上到下，顺序执行
    3. # 表示注释
    4. 每条指令都会创建一个新的镜像层，并对镜像进行提交
4. dockerfile 执行流程
    1. docker 从基础镜像运行一个容器
    2. 执行一条指令并对容器作出修改
    3. 执行类似docker commit 的操作提交一个新的镜像层
    4. docker 在基于 更提交的景象运行一个新容器
    5. 执行dockerfile中的下一条指令
5. docker 是软件源码
    1. docker 镜像是软件的交付品
    2. docker 容器则可以认为软件的运行态

6. 关键字
关键字 参数 说明
FROM 表示该镜像基于哪个镜像
MAINTAINER 作者姓名与作者邮箱
RUN 容器构建时需要运行的命令
EXPOSE 当前容器的端口号
WORKDIR 容器启动后的工作目录 相当于 cd /xxx
ENV 用来构建镜像时设置环境变量
ADD 将主机目录下的文件拷贝加解压缩， 自动处理URL 和 解压tar压缩包
COPY 将主机目录下的文件拷贝镜像内
VOLUME 容器数据卷，用户数据保存和持久化工作，并会在 /var/lib/docker/volumes/, 使用 inspect 容器进行查看 source
CMD 指定一个容器启动时需要运行的命令，shell 格式 和 exec 格式， CMD 会被docker run之后的参数替换
ENTRYPOINT 与 CMD 一样，但ENTRYPOINT可以执行多条语句
ONBUILD 当构建继承时运行改名，父镜像在子镜像继承后的，onbuild 被触发
USER 对用户权限进行设置
